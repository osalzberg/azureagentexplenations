 Scoring Rubric

                    
                

                

                    

                        

                            
 Faithfulness (25%)

                            
No hallucinated metrics, causes, tables, or columns

                            
    
                                
  • 5: Perfectly accurate, all claims supported by data
    • 
                                
  • 3: Mostly accurate, minor unsupported inferences
    • 
                                
  • 1: Contains hallucinated data or false claims
    • 
                            

                        

                        

                            
 Structure (10%)

                            
Logical sections, headings, good flow

                            
    
                                
  • 5: Clear sections, logical flow, easy to scan
    • 
                                
  • 3: Some structure, could be better organized
    • 
                                
  • 1: Wall of text, no clear organization
    • 
                            

                        

                        

                            
 Clarity (15%)

                            
Explains KQL and Azure concepts clearly

                            
    
                                
  • 5: Crystal clear, accessible to target audience
    • 
                                
  • 3: Understandable but some jargon unexplained
    • 
                                
  • 1: Confusing, assumes too much knowledge
    • 
                            

                        

                        

                            
 Analytical Depth (20%)

                            
Explains WHY the results matter

                            
    
                                
  • 5: Deep insights, patterns, implications explained
    • 
                                
  • 3: Some analysis beyond restating numbers
    • 
                                
  • 1: Just restates the data with no insight
    • 
                            

                        

                        

                            
 Azure Context (15%)

                            
Correct interpretation of Azure telemetry

                            
    
                                
  • 5: Expert-level Azure/App Insights knowledge
    • 
                                
  • 3: Correct but basic Azure understanding
    • 
                                
  • 1: Wrong interpretation of Azure concepts
    • 
                            

                        

                        

                            
 Actionability (10%)

                            
Relevant, realistic next steps

                            
    
                                
  • 5: Specific, actionable, aligned with data
    • 
                                
  • 3: Generic but relevant recommendations
    • 
                                
  • 1: No recommendations or irrelevant ones
    • 
                            

                        

                        

                            
 Conciseness (5%)

                            
No fluff, no major omissions

                            
    
                                
  • 5: Perfect balance, every sentence adds value
    • 
                                
  • 3: Slightly verbose or missing minor details
    • 
                                
  • 1: Extremely verbose or missing key info
    • 
                            

                        

                    

                

            


            
            

                

                    
 Failure Examples

                    
                

                

                    

                        

                            
Critical

                            
Hallucinating Root Causes

                            
"The high error rate is caused by a database connection timeout issue in the payment service."

                            
❌ Data shows errors but no evidence of database issues or payment service involvement.

                        

                        

                            
Critical

                            
Misinterpreting Azure Semantics

                            
"ResultCode 0 indicates a successful request."

                            
❌ In HTTP context, ResultCode 0 typically means no response/timeout, not success (200).

                        

                        

                            
Poor

                            
Restating Without Insight

                            
"The query returned 5 rows. Row 1 has value 100, row 2 has value 50..."

                            
❌ Merely restates numbers without explaining what they mean or why they matter.

                        

                        

                            
Poor

                            
Generic Recommendations

                            
"Consider monitoring your application and setting up alerts."

                            
❌ Generic advice not tied to the specific data or findings.
